# 역할별 분석 체크리스트

에이전트 프롬프트의 `## 분석 체크리스트`에 해당 역할 섹션만 추출하여 삽입한다.

## 보안
- OWASP Top 10 (인젝션, 인증 우회, XSS, CSRF, SSRF)
- 인증/인가: 토큰 검증, 세션 관리, 권한 상승
- 시크릿 관리: 하드코딩된 키/토큰/비밀번호
- CORS 설정, CSP 헤더
- 입력 검증 및 출력 인코딩
- 의존성 취약점 (알려진 CVE)
- 탐색 힌트: `.env`, `*config*`, `*auth*`, `*secret*` 파일 우선. `grep -rn "password\|secret\|api.key\|token" --include="*.{ts,py,go,js}"` 실행

## 코드 품질
- DRY 원칙 위반 (중복 코드)
- 순환 복잡도 과다 (깊은 중첩, 긴 함수)
- 에러 처리: 누락, 삼킨 에러, 불충분한 에러 메시지
- 타입 안전성: any 남용, 제네릭 캐스팅으로 우회
- 데드 코드, 미사용 import/변수
- 네이밍 일관성

## 성능
- N+1 쿼리 패턴
- 캐싱 부재 또는 무효화 누락
- 인덱스 누락 (자주 조회하는 컬럼)
- 번들 크기: 불필요한 의존성, tree-shaking 실패
- 비동기 처리: 병렬 가능한데 순차 실행
- 리소스 정리: 커넥션/파일핸들/타이머 미해제
- 메모리 누수 패턴
- 탐색 힌트: `*query*`, `*fetch*`, `*cache*` 파일. `grep -rn "await\|\.then\|Promise\.all" --include="*.{ts,js}"` + DB 쿼리 패턴 검색

## 아키텍처
- 레이어 분리: 비즈니스 로직이 컨트롤러/UI에 혼재
- API 설계: 일관성, 버저닝, 에러 응답 형식
- 서비스 경계: 순환 의존, 과도한 결합
- 확장성: 단일 장애 지점, 수평 확장 가능성

## 프론트엔드
- 컴포넌트 구조: 과대 컴포넌트, 책임 혼재
- 상태 관리: prop drilling, 불필요한 전역 상태
- 접근성: ARIA, 키보드 네비게이션, 색상 대비
- 반응형: 브레이크포인트, 모바일 레이아웃
- 탐색 힌트: `src/components/`, `src/hooks/`, `src/app/` 우선. 가장 큰 컴포넌트 파일부터 확인

## DB
- 스키마 설계: 정규화, 관계 무결성
- 인덱스: 복합 인덱스, 커버링 인덱스
- 마이그레이션: 안전성, 롤백 가능성
- 쿼리 최적화: 풀 스캔, 비효율적 JOIN
- 탐색 힌트: `migrations/`, `schema/`, `models/`, `prisma/` 디렉토리 우선. ORM 설정 파일 확인

## 테스트
- 테스트 피라미드: 유닛/통합/E2E 비율
- 커버리지: 핵심 비즈니스 로직 포함 여부
- 엣지 케이스: 경계값, null/빈값, 동시성
- CI 통합: 테스트 자동 실행, 실패 시 블로킹

## AI/ML
- 프롬프트 설계: 인젝션 방어, 구조화
- 모델 통합: API 키 관리, 폴백, 타임아웃
- 평가 메트릭: 정확도/재현율 측정 체계
- 안전 가드레일: 출력 필터링, 콘텐츠 정책
- 탐색 힌트: `*prompt*`, `*llm*`, `*ai*`, `*agent*` 파일. `grep -rn "openai\|anthropic\|claude\|gpt" --include="*.{ts,py}"` 실행

## 인프라/배포
- IaC: 코드로 관리되지 않는 인프라
- CI/CD: 빌드/테스트/배포 파이프라인 완성도
- 컨테이너: 이미지 크기, 멀티스테이지 빌드
- 롤백: 배포 실패 시 복구 절차
- 비용 최적화: 과잉 프로비저닝
- 탐색 힌트: `Dockerfile`, `.github/workflows/`, `terraform/`, `docker-compose*`, `*deploy*` 파일 우선

## 디버깅
- 에러 패턴: 반복되는 에러, 근본 원인 미해결
- 로깅: 구조화된 로그, 적절한 로그 레벨
- 레이스 컨디션: 동시 접근, 락 누락
- 리그레션: 이전 수정이 되돌아간 패턴
- 탐색 힌트: `grep -rn "catch\|except\|error\|throw\|panic" --include="*.{ts,py,go}"` + 로그 설정 파일 확인

## 문서
- API 문서: 엔드포인트, 파라미터, 응답 형식
- 아키텍처 문서: 시스템 구조, 데이터 흐름
- 온보딩 가이드: 개발 환경 설정, 실행 방법

## UI/UX
- 사용성: 직관적 인터페이스, 학습 곡선
- 접근성: 스크린 리더, 키보드 전용 사용
- 디자인 일관성: 컬러/폰트/간격 통일
- 에러 UX: 사용자 친화적 에러 메시지, 복구 경로

## 장애 대응
- 장애 분류: 심각도 정의, 에스컬레이션 경로
- 모니터링: 알림 설정, 대시보드
- 복구 절차: 런북, 자동 복구
- 포스트모템: 근본 원인 분석 체계

## 언어 전문가
- 관용적 코드: 해당 언어의 베스트 프랙티스
- 타입 시스템: 언어 고유 타입 기능 활용
- 생태계 활용: 표준 라이브러리, 커뮤니티 패키지
